# Push to induscorporations/snapsy – Account switch steps

## Done already
- Remote `origin` is set to: `https://github.com/induscorporations/snapsy.git`
- Stored GitHub credential (previous account) has been removed from Windows Credential Manager

## What you need to do

### 1. Use the correct GitHub account
- You must push using an account that has **write access** to `induscorporations/snapsy` (e.g. an induscorporations org member or the repo owner).

### 2. GitHub no longer accepts account password over HTTPS
- When Git asks for a password, use a **Personal Access Token (PAT)**, not your GitHub password.
- Create a token: https://github.com/settings/tokens → “Generate new token (classic)” → enable at least **repo**.
- Copy the token and keep it somewhere safe (you won’t see it again).

### 3. (Optional) Set Git user for this repo
If you want commits to show the new account’s name/email in this repo only:

```powershell
cd D:\Indus\snapsy
git config user.name "Your New Name"
git config user.email "your-new-account@example.com"
```

### 4. Push (you’ll be prompted to sign in)
From the project folder:

```powershell
cd D:\Indus\snapsy
git push -u origin master
```

- **Username:** your GitHub username (the account that has access to induscorporations/snapsy).
- **Password:** paste your **Personal Access Token** (not your GitHub password).

Windows will store this new credential for future pushes.

### 5. If you need to switch accounts again later
Remove the stored credential, then push again:

```powershell
cmdkey /delete:LegacyGeneric:target=git:https://github.com
git push -u origin master
```

Then sign in with the desired account (username + PAT).
